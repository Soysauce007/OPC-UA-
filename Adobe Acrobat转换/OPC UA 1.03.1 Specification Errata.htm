2 OPC UA Specification: Part 5 – Information Model

Topic

Improve the description for the situation where collection of diagnostic information is disabled.

Errata Version

1.03.1

Spec Reference

Part 5

Clause 6.3.3 – ServerDiagnosticsType, last paragraph

Mantis Reference

0003219

Problem Statement

The current description is ambiguous about what nodes have to exist in the AddressSpace if the EnabledFlag is FALSE (collection disabled).

Solution

Add the following statement:

When diagnostics are turned off, the Server can return Bad_NodeIdUnknown for all static diagnostic Nodes except the EnabledFlag Property. Dynamic diagnostic Nodes (such as the Session Nodes) will not appear in the AddressSpace.

If collection of diagnostic information is not supported at all, the EnabledFlag Property will be ReadOnly.

It replaces the following paragraph:

Static diagnostic Nodes that always appear in the AddressSpace will return Bad_NotReadable when the Value Attribute of such a Node is read or subscribed to and diagnostics are turned off. Dynamic diagnostic Nodes (such as the Session Nodes) will not appear in the AddressSpace when diagnostics are turned off.
